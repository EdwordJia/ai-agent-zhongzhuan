# OpenClaw 绿灯测试案例

> 绿灯测试（Green Light Tests）是每次功能开发完成后必须跑通的最小业务验收集。通过即表示该功能在当前分支上可进入人类审核。

## 执行约定

- **自动化案例**：纳入 `pnpm run test:e2e`，每次 build 前自动运行。
- **手动/半自动案例**：标注为 `[MANUAL]`，需要真实 LLM / 付费接口，仅在业务冒烟测试或发布前由开发者手动触发。
- **新增功能**：开发者需在本文档补充对应绿灯案例，并明确其执行方式。
- **失败处理**：任一绿灯案例未通过，禁止提交人类审核或合并。

---

## 1. 核心页面渲染

### GL-01 仪表盘正常加载

- **触发**：访问 `/`
- **期望**：
  - 页面标题为"仪表盘"
  - 显示商品总数、素材图片、AI 任务、导出任务卡片
  - 控制台无未捕获异常
- **执行**：Playwright 自动化（`e2e/smoke.spec.ts::dashboard renders`）

---

## 2. 商品管理

### GL-02 商品 CRUD

- **触发**：
  1. 进入 `/products/new` 创建商品
  2. 在列表中编辑商品价格
  3. 删除商品
- **期望**：
  - 新建成功后跳转列表并可见
  - 编辑保存后价格更新
  - 删除后列表为空提示出现
- **执行**：Playwright 自动化（`e2e/smoke.spec.ts::product CRUD`）

---

## 3. AI 文案生成

### GL-03 AI 生成并应用到商品

- **触发**：
  1. 创建测试商品
  2. 进入 `/ai-generate`
  3. 选择商品、输入关键词、点击开始生成
  4. 点击"应用到商品"
- **期望**：
  - 显示"生成成功"
  - 点击应用后提示"标题已应用到商品"
- **执行**：Playwright 自动化（`e2e/smoke.spec.ts::AI generate`）

---

### GL-08 AI 生图生成单张图片 [MANUAL]

  3. 选择生成数量为 8
  4. 点击"开始生图"
- **期望**：

  - 最终展示 8 张不同图片的网格列表
  - 每张图片可单独保存/复制
- **执行**：手动，依赖真实 gpt-image-2 接口

## 8. 智能体聊天

### GL-10 聊天框基础交互

- **触发**：
  1. 在首页点击右下角聊天按钮
  2. 输入问候语并发送
  3. 关闭聊天框
- **期望**：
  - SideSheet 正常打开/关闭
  - 用户消息和助手回复均可见
- **执行**：Playwright 自动化（`e2e/smoke.spec.ts::chatbox opens and responds`，当前已 mock LLM）

### GL-11 聊天框通过真实 LLM 回复 [MANUAL]

- **触发**：
  1. 确保 Settings 中 AI 配置有效（默认已写死 newapi.panda-w.com）
  2. 打开聊天框
  3. 发送"你好，介绍一下你能做什么"
- **期望**：
  - 收到真实 LLM 回复，内容包含 OpenClaw 能力介绍
  - 非关键词兜底文案
- **执行**：手动，依赖真实 LLM 接口

### GL-12 聊天框通过 MCP 调用商品列表 [MANUAL]
